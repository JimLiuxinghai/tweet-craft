Notion 配置教程

一次性把 Token、页面授权、数据库配置理顺。完成后点击推文的 Notion 按钮即可保存媒体、正文和链接。

预计 6 分钟 支持 secret_ / ntn_ 已有数据库可复用

开始前准备

  • 1Notion 账号 + 可编辑的工作区
  • 2浏览器已安装 Tweet Craft 扩展
  • 3准备一个父页面（用来放数据库）
  • 4可复用现成数据库（可选）

快速流程

按顺序完成四步，最常见的问题都能避免。

01

创建集成

访问 Notion 集成页面，创建 Internal integration，拿到 Token。

02

授权父页面

在 Notion 页面右上角分享给集成，否则无法读写数据库。

03

连接扩展

在扩展的 Notion 设置中粘贴 Token，点击连接。

04

复用或新建数据库

已有数据库会优先复用；否则选择父页面创建。

详细步骤

更适合第一次配置或者遇到权限提示时逐项核对。

1

创建 Integration

打开 https://www.notion.so/my-integrations → “+ New integration” → 选择 Internal integration。

  • 确保 Token 以 secret_ 或 ntn_ 开头。
  • 保存 Token，后面要粘贴到扩展。
2

把集成分享给页面

打开你准备用来放数据库的页面，右上角 “•••” → Add connections → 选择集成。

  • 没有授权就会出现 “权限不足 / 无可用页面”。
3

连接扩展

进入扩展 → Notion 设置 → 粘贴 Token → 点击 “连接 Notion”。

  • 连接成功会显示 “Notion 已连接”。
4

复用或创建数据库

点击 “配置数据库”：

  • 只有 1 个数据库时会自动复用。
  • 多数据库时会弹出选择器。
  • 无数据库时选择父页面创建。
5

保存测试

在 X/Twitter 里点击推文的 Notion 按钮，检查数据库是否出现内容与媒体信息。

数据库 ID 怎么找

数据库 ID 可选，不填也能创建新库；如果你要手动绑定，按下面方法取值。

从 URL 里截取

打开数据库页面，URL 中第一段 32 位字符串就是数据库 ID（去掉中划线也可）。

示例格式

https://www.notion.so/aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa?v=...

常见问题

先看这里，能解决 90% 的连接失败。

提示 “Integration Token 无效或已过期”

重新复制完整 Token，确认以 secret_ 或 ntn_ 开头，并没有空格。

提示 “权限不足 / 没有可用页面”

回到 Notion 页面右上角 “Add connections”，把集成加进去。

数据库信息为空

检查是否有多个工作区，确保集成授权的是当前页面所在工作区。

媒体信息没有保存

确认推文有图片/视频，扩展设置里允许保存媒体；保存后检查 “媒体信息 / 媒体摘要” 字段。

需要快速自检？在扩展里点击 “测试连接 / 运行诊断”。
